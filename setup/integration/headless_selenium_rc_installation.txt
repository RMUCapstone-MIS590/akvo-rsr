Installation and setup for running a Selenium RC server in headless mode
------------------------------------------------------------------------

This installation will allow TeamCity build agents to run Selenium tests on a headless Linux server.

 1) Create a clone of the http://github.com/akvo/akvo-rsr repository in a temporary directory
 2) Create the /usr/local/ci/scripts directory if necessary
 3) From the akvo-rsr repository, copy the following files to the /usr/local/ci/scripts directory:
      sudo cp -p scripts/testing/start* /usr/local/ci/scripts
      sudo cp -p scripts/testing/stop* /usr/local/ci/scripts
      sudo cp -p scripts/testing/ensure* /usr/local/ci/scripts
 4) Change their ownership to root:
      sudo chown root:root /usr/local/ci/scripts/*
 5) Copy the etc_init.d_selenium_rc.sh file to the /etc/init.d directory:
      sudo cp -p setup/integration/scripts/etc_init.d_selenium_rc.sh /etc/init.d/selenium_rc
 6) Make the script executable and change its ownership to root:
      sudo chmod 755 /etc/init.d/selenium_rc
      sudo chown root:root /etc/init.d/selenium_rc
 7) Link the startup script to the startup directories:
      sudo ln -s /etc/init.d/selenium_rc /etc/rc1.d/K99selenium_rc
      sudo ln -s /etc/init.d/selenium_rc /etc/rc2.d/S99selenium_rc
 8) Reboot the server so that the headless Selenium RC server instance will start automatically:
      sudo reboot
 9) Once the server has restarted, check that the Xvfb and Selenium RC processes are running on the expected display
